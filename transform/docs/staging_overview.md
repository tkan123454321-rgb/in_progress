### Staging Layer Processing Logic

The Staging layer serves as the **Technical Standardization** zone. The primary goal here is to clean the raw data formats without applying any business or filtering logic. The data grain remains exactly the same as the Bronze layer.

**Key Transformation Steps:**
1. **Naming Conventions:** Standardize all column names to `snake_case`, removing any leading/trailing whitespaces and special characters.
2. **Type Casting:** Convert raw string values into appropriate data types.
3. **JSON Parsing (if applicable):** Extract nested properties from raw JSON strings into dedicated physical columns.
4. **Audit Columns:** Inject metadata columns generated by dbt to allow data lineage tracking.

*Note: No data rows are filtered (dropped) in this layer. Bad data is passed through to be caught and handled in the Silver layer.*
